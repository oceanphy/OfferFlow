# 0001 — 面试回合作为诊断最小单元

诊断以"面试回合"而非"Q&A 对"为最小单元。一个回合包含一个话题加该话题下的所有追问，诊断时作为整体处理。

**Considered Options**

1. **Q&A 对** — 每次一问一答独立诊断。简单直接，但追问链会被拆散，后续回答脱离上下文后诊断质量下降。
2. **整场面试** — 一场面试作为一个整体诊断。省去了拆分步骤，但粒度太粗，问题定位无法精确到具体话题。
3. **面试回合** — 话题加追问链作为整体。拆分和合并逻辑复杂，但诊断时上下文完整，问题定位精准。

选择方案 3。面试中追问是常态，追问的回答脱离原问题上下文无法独立评估。回合模型牺牲了拆分的一致性，换取了诊断质量。

**Consequences**

- `split_rounds` tool 的拆分逻辑需要识别话题边界和追问模式，比简单按"问-答"拆分更复杂。
- 后续扩展到实时语音转写时，回合边界更模糊，需要重新审视拆分策略。
